# Typetchi v0.6.3

Version: 0.6.3

## Collapsed Handle Layout / Drag Bounds Bugfix

### Problem

Collapsed mode still kept the full widget panel box around the small icon, which could leave a large transparent / blurred hitbox. Drag bounds also used the expanded widget size, so the collapsed handle could only be dragged through part of the viewport.

### Fix

- Collapsed mode now sizes the widget container to the handle footprint only.
- Collapsed mode removes panel blur, panel shadow, border, and background.
- Collapsed mode hides the expanded header, body, resize control, footer, and toast.
- Drag bounds use collapsed handle dimensions while collapsed, and expanded widget dimensions while expanded.
- No evolution, EXP, paste, or popup close/reopen behavior was changed.

### Acceptance checks

- Collapse shows only the small handle/icon.
- No large blurred panel or transparent hitbox remains while collapsed.
- The collapsed handle can be dragged near the bottom and right viewport edges.
- Dragging the handle persists the new position.
- Clicking the handle without dragging still expands the widget.
