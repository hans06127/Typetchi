# Typetchi v0.6.4

Version: 0.6.4

## Collapsed Handle Drag Bottom Bounds Bugfix

### Problem

v0.6.3 updated drag-time bounds to use the collapsed handle size, but widget state normalization still clamped saved and remote `x` / `y` with the expanded widget width and height. That meant a handle dragged near the viewport bottom could be saved or synchronized, then normalized back upward as if the full widget had to fit there.

### Fix

- Split position normalization semantics by state.
- Expanded widget normalization uses the full widget width and height.
- Collapsed handle normalization uses the collapsed handle footprint (`64 × 52`).
- Drag-time bounds and save/load/remote normalization now use the same collapsed handle dimensions.
- Position storage remains viewport `x` / `y` coordinates.
- No evolution, EXP, paste, CPM/WPM, daily stats, close, or popup reopen behavior was changed.

### Acceptance checks

- Collapsed handle can be dragged near the viewport bottom and four corners.
- Releasing after bottom drag does not snap the handle back upward.
- Reloading while collapsed preserves the bottom handle position.
- Cross-tab sync preserves the collapsed handle position instead of applying expanded widget bounds.
- Clicking the handle still expands; dragging the handle does not expand.
