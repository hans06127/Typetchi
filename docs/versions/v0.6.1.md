# Typetchi v0.6.1 Bugfix：Close / Collapse 修正與 v0.6.0 規格補齊確認

## 1. 版本目標

v0.6.1 是 v0.6.0 之後的 bugfix 版本。

目前發現兩個問題：

1. Widget header 上出現兩個「收合」按鈕。
2. 原本應該保留的「關閉」功能被改成收合，導致 UI 語意錯誤。
3. 使用者期望「關閉」仍然存在，但關閉後要能透過 Chrome 擴充套件圖示重新開啟。
4. 需要確認原定 v0.6.0 的「進化系統重構與分支進化預留」是否真的有被實作進專案。

本版本先修正 bug 與補齊驗收，不新增新玩法。

---

## 2. 目前畫面問題

目前 Widget header 顯示：

```txt
Typetchi    固定    收合    收合
```

這是不正確的。

正確應該是：

```txt
Typetchi    固定    收合    關閉
```

或使用 icon：

```txt
Typetchi    固定    －    ×
```

但按鈕語意必須清楚區分：

```txt
收合 = collapse
關閉 = close / hide
```

---

## 3. 正確行為定義

## 3.1 收合 collapse

收合是暫時縮小 Widget。

行為：

```txt
點擊「收合」
→ Widget 向下收合
→ 保留 handle / bubble / 小頭像
→ 點擊 handle 可展開
→ collapsed = true
```

特性：

```txt
可跨分頁同步
可持久化
重新整理後可以維持收合狀態
使用者仍看得到 handle
不會完全消失
```

---

## 3.2 關閉 close / hide

關閉是完全隱藏 Widget。

行為：

```txt
點擊「關閉」
→ Widget 完全隱藏
→ 不顯示完整 panel
→ 不顯示 handle
→ 點擊 Chrome 擴充套件圖示 / popup 後可以重新開啟
```

特性：

```txt
可以跨分頁同步
可以持久化
必須提供恢復入口
不得讓使用者永遠打不開
```

---

## 4. 狀態設計

請不要混用 collapsed 與 closed。

建議明確拆分：

```ts
export interface WidgetState {
  x: number | null;
  y: number | null;
  width: number;
  height: number;
  pinned: boolean;
  collapsed: boolean;
  closed: boolean;
}
```

語意：

```txt
collapsed = true
表示 Widget 收合，但 handle 仍顯示。

closed = true
表示 Widget 完全隱藏，handle 也不顯示。
```

---

## 5. Render 規則

### 5.1 closed = true

如果：

```ts
widgetState.closed === true
```

則 Content Script 中的 Widget 可以不渲染 panel 與 handle。

但 root / message listener / storage sync 必須仍然存在，確保可收到 popup 重新開啟訊息。

行為：

```txt
Root exists
React app exists
Widget visual hidden
仍可接收 storage.onChanged
仍可接收 runtime message
```

不得把整個 content script runtime 停掉。

---

### 5.2 collapsed = true

如果：

```ts
widgetState.collapsed === true
```

則顯示收合 handle。

行為：

```txt
Widget panel hidden / 向下收合
Handle visible
Click handle → collapsed = false
```

---

### 5.3 closed 優先權

如果同時：

```ts
closed === true
collapsed === true
```

則以 `closed` 為優先。

也就是：

```txt
closed true → 完全隱藏
```

重新開啟時建議：

```ts
closed = false
collapsed = false
```

---

## 6. Header 按鈕修正

請確認 Header 只顯示三個主要操作：

```txt
固定 / 解除固定
收合
關閉
```

不得出現兩個收合。

### 6.1 文案建議

```txt
固定
收合
關閉
```

如果已固定：

```txt
解除固定
收合
關閉
```

### 6.2 Tooltip 建議

```txt
固定視窗
收合小視窗
關閉 Typetchi
```

## 7. 關閉按鈕流程

```ts
async function handleCloseWidget() {
  const nextWidgetState = {
    ...widgetState,
    closed: true,
  };

  setWidgetState(nextWidgetState);
  await saveWidgetState(nextWidgetState);
}
```

若有跨分頁同步，其他分頁也應收到 `closed: true` 並隱藏 Widget。

---

## 8. 收合按鈕流程

```ts
async function handleCollapseWidget() {
  const nextWidgetState = {
    ...widgetState,
    collapsed: true,
    closed: false,
  };

  setWidgetState(nextWidgetState);
  await saveWidgetState(nextWidgetState);
}
```

---

## 9. Handle 展開流程

```ts
async function handleExpandWidget() {
  const nextWidgetState = {
    ...widgetState,
    collapsed: false,
    closed: false,
  };

  setWidgetState(nextWidgetState);
  await saveWidgetState(nextWidgetState);
}
```

# 10. Chrome 擴充套件圖示重新開啟

## 10.1 需求

當 Widget 被關閉後，使用者需要可以點 Chrome 擴充套件圖示重新開啟。

最低要求：

```txt
點擊 Chrome extension icon
→ 開啟 popup
→ popup 中有「顯示 Typetchi」按鈕
→ 點擊後把 widgetState.closed 改成 false
→ 所有已開啟分頁同步顯示 Widget
```

---

## 10.2 Manifest 檢查

請確認 `manifest.json` 有 popup：

```json
{
  "action": {
    "default_popup": "popup.html",
    "default_title": "Typetchi"
  }
}
```

如果目前沒有 popup，請在 v0.6.1 新增簡單 popup。

---

## 10.3 Popup 最低 UI

Popup 不需要漂亮，v0.6.1 只需要能恢復 Widget。

建議內容：

```txt
Typetchi

目前狀態：顯示中 / 已關閉

[顯示 Typetchi]
```

如果已顯示中，按鈕可以顯示：

```txt
已顯示
```

或仍允許點擊，將：

```txt
closed = false
collapsed = false
```

---

## 10.4 Popup 檔案建議

```txt
src/popup/index.html
src/popup/index.tsx
src/popup/Popup.tsx
src/popup/Popup.module.scss
```

如果專案沒有 React popup，也可以用簡單原生 JS。

---

## 10.5 Popup 重新開啟流程

```ts
async function handleShowTypetchi() {
  const widgetState = await loadWidgetState();

  const nextWidgetState = {
    ...widgetState,
    closed: false,
    collapsed: false,
    updatedAt: Date.now(),
  };

  await saveWidgetState(nextWidgetState);
}
```

依靠 v0.5.0 的 `chrome.storage.onChanged` 讓所有分頁同步顯示。

若部分分頁 content script 沒有即時收到 storage change，popup 可額外透過 runtime message 通知 tabs，但最低要求是 storage sync 正常。

---

## 11. Popup 與無開啟分頁情境

如果目前沒有任何可注入頁面開著，popup 只要把 storage 設成：

```txt
closed = false
collapsed = false
```

即可。

下一次使用者開啟一般網頁時，Typetchi 應正常顯示。

---

## 12. Storage normalize 規則更新

v0.6.1 不再一律把 `closed` normalize 成 false，否則關閉就無法持久化。

請改成：

```txt
closed 可以持久化
但必須有 popup 恢復入口
```

Normalize 只處理非法型別：

```ts
closed: typeof raw.closed === 'boolean' ? raw.closed : false
collapsed: typeof raw.collapsed === 'boolean' ? raw.collapsed : false
```

但若沒有完成 popup，就不得持久化 `closed: true`。

---

## 13. 跨分頁同步需求

### 13.1 關閉同步

```txt
A 分頁點擊關閉
→ A 分頁隱藏 Widget
→ B 分頁同步隱藏 Widget
→ C 分頁同步隱藏 Widget
```

### 13.2 Popup 開啟同步

```txt
點擊 Chrome extension icon
→ popup 點擊「顯示 Typetchi」
→ A / B / C 分頁同步顯示 Widget
```

### 13.3 收合同步

```txt
A 分頁點擊收合
→ A 分頁保留 handle
→ B 分頁保留 handle
→ 任一分頁點擊 handle 展開
→ 所有分頁展開
```

# 14. v0.6.0 原定規格補齊檢查

## 14.1 需求

使用者懷疑原定 v0.6.0 規格沒有被真正寫進專案。

請檢查並回報 v0.6.0 是否已完成以下項目。

---

## 14.2 必查項目

請確認是否存在或已實作：

```txt

src/config/evolutionNodes.ts
src/systems/evolutionSystem.ts
src/systems/evolutionProgressSystem.ts
src/storage/petStateMigration.ts
```

若專案檔案名稱不同，請回報實際位置。

---

## 14.3 必查功能

請確認：

1. 是否有 `EvolutionNode` 型別。
2. 是否有 `EvolutionCondition` 型別。
3. 是否有 `evolutionNodes` config。
4. 是否已用 config 驅動三階段進化。
5. 是否還存在硬寫 `if totalExp >= 2000 return stage_3` 作為主要邏輯。
6. 是否有 `currentEvolutionNodeId`。
7. 是否有舊 petState migration。
8. EXP bar 是否已改用 evolution config。
9. UI 是否顯示下一階段名稱與條件。
10. 重置角色進度是否重置 `currentEvolutionNodeId`。
11. 跨分頁同步是否包含 `currentEvolutionNodeId`。

---

## 14.4 若 v0.6.0 未完成

如果上述項目缺失，請不要只修 close / popup。

請同時補齊 v0.6.0 原定規格中尚未完成的 evolution system 重構。

優先順序：

```txt
1. 先修 close / collapse / popup 開啟
2. 再確認版本號與 manifest
3. 再補齊 v0.6.0 evolution system
4. 最後完整回歸測試
```

# 16. 測試案例

## 16.1 Header 按鈕

1. 開啟 Widget。
2. Header 不得出現兩個「收合」。
3. Header 應有「收合」與「關閉」兩個不同操作。
4. Tooltip / title 不得誤導。

---

## 16.2 收合

1. 點擊「收合」。
2. Widget 向下收合。
3. Handle 保留。
4. 點擊 handle 可以展開。
5. 跨分頁同步正常。

---

## 16.3 關閉

1. 點擊「關閉」。
2. Widget 完全隱藏。
3. Handle 不顯示。
4. 重新整理頁面後若 closed 持久化，仍保持關閉。

---

## 16.4 Popup 重新開啟

1. 點擊 Chrome extension icon。
2. Popup 顯示「顯示 Typetchi」。
3. 點擊後 Widget 重新顯示。
4. 所有已開啟分頁同步重新顯示。
5. `closed = false`。
6. `collapsed = false`。

---

## 16.5 v0.6.0 evolution 檢查

4. EXP 2000 進化到 stage_3。
