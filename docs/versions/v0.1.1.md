# Typetchi v0.1.1 修正規格

## 1. 版本目標

v0.1.1 是 Typetchi 的穩定性修正版。

本版本不新增遊戲玩法，不新增角色系統功能，主要目標是讓 Chrome Extension 在不同網站中都能穩定注入小視窗，並確保 Typetchi 不影響宿主網站原本功能。

---

## 2. 目前問題

目前 v0.1.0 測試時發現：

1. 有些網站重新整理後，右下角小視窗會短暫出現後消失。
2. 有些網站可以正常顯示小視窗。
3. Console 可能出現以下錯誤：

```txt
Uncaught (in promise) Error: A listener indicated an asynchronous response by returning true, but the message channel closed before a response was received
```

```txt
RecoverableError: Minified React error #418
```

因此判斷目前 Extension 基本注入流程已經有執行，但在部分網站會因為 React 掛載方式、DOM 注入方式、樣式污染、重複注入或 message listener 處理不完整而導致 UI 消失或錯誤。

---

## 3. 本版本核心任務

### 3.1 修正 React 掛載方式

請檢查 Content Script 是否使用 `hydrateRoot`。

Chrome Extension Content Script 注入 UI 不應使用 `hydrateRoot`，請統一改成 `createRoot`。

請使用：

```ts
import { createRoot } from 'react-dom/client';

createRoot(rootElement).render(<App />);
```

不得使用：

```ts
hydrateRoot(rootElement, <App />);
```

---

### 3.2 防止重複注入

Content Script 注入前必須檢查是否已存在 root。

Root id 固定為：

```txt
typetchi-root
```

若已存在，請不要重複建立，也不要重複掛載 React。

範例：

```ts
const ROOT_ID = 'typetchi-root';

function injectTypetchiRoot() {
  const existingRoot = document.getElementById(ROOT_ID);

  if (existingRoot) {
    console.log('[Typetchi] root already exists, skip injection');
    return;
  }

  const root = document.createElement('div');
  root.id = ROOT_ID;
  document.body.appendChild(root);

  createRoot(root).render(<App />);
}
```

---

### 3.3 等待 document.body

如果 `document.body` 尚未存在，請等待 `DOMContentLoaded` 後再注入。

```ts
if (document.body) {
  injectTypetchiRoot();
} else {
  window.addEventListener('DOMContentLoaded', injectTypetchiRoot, { once: true });
}
```

---

## 4. 使用 Shadow DOM 隔離 UI

### 4.1 目的

Typetchi 是注入到宿主網站內的 UI，因此必須避免：

1. 宿主網站 CSS 影響 Typetchi UI。
2. Typetchi CSS 影響宿主網站。
3. 宿主網站的 reset style 造成小視窗消失或變形。
4. Typetchi 的樣式污染原頁面。

### 4.2 DOM 結構

請將 React App 掛載到 Shadow DOM 內。

建議結構：

```txt
document.body
  #typetchi-root
    shadowRoot
      #typetchi-app
```

### 4.3 限制

Typetchi 的 CSS 不得直接插入：

```txt
document.head
```

所有 Typetchi UI 樣式應只存在於 shadowRoot 內。

---

## 5. Root 樣式規範

`#typetchi-root` 必須使用固定定位，並確保不影響頁面布局。

建議：

```ts
root.style.position = 'fixed';
root.style.right = '24px';
root.style.bottom = '24px';
root.style.zIndex = '2147483647';
root.style.width = '280px';
root.style.height = '360px';
root.style.pointerEvents = 'none';
root.style.contain = 'layout style paint';
```

真正的小視窗容器才允許接收滑鼠事件：

```ts
widget.style.pointerEvents = 'auto';
```

目的：

```txt
透明區域不得擋住原網站點擊。
只有 Typetchi 小視窗本體可以被點擊、拖曳、縮放。
```

---

## 6. 不干擾宿主網站規範

Typetchi 不得影響任何頁面原本功能。

### 6.1 禁止行為

禁止修改宿主頁面的既有 DOM：

```txt
不得修改 document.body.innerHTML
不得修改原頁面元素 class
不得修改原頁面元素 style
不得移除原頁面元素
不得包裹原頁面元素
不得將 Typetchi CSS 注入 document.head
```

禁止覆寫宿主頁面全域物件：

```txt
不得覆寫 window
不得覆寫 document
不得覆寫 localStorage
不得覆寫 sessionStorage
不得覆寫 fetch
不得覆寫 XMLHttpRequest
不得覆寫 console
不得覆寫 EventTarget.prototype.addEventListener
```

禁止攔截宿主網站事件：

```txt
不得對宿主網站事件呼叫 event.preventDefault()
不得對宿主網站事件呼叫 event.stopPropagation()
不得阻止 input / textarea / contenteditable 原本輸入行為
不得阻止頁面滾動
不得阻止頁面快捷鍵
```

例外：

如果事件發生在 Typetchi 自己的 Shadow DOM UI 內，可以處理拖拉、縮放、按鈕點擊等事件。

## 7. 鍵盤與輸入監聽規範

### 7.1 監聽方式

第一版請優先使用 `input` event 監聽有效輸入，不要使用全域 `keydown` 計算字數。

```ts
document.addEventListener('input', handleInput, true);
```

### 7.2 不得阻止事件

`handleInput` 不得呼叫：

```ts
event.preventDefault();
event.stopPropagation();
```

Typetchi 只做旁路統計，不干涉頁面原本功能。

## 8. 表單安全與隱私規範

### 8.1 第一版允許統計的欄位

第一版建議只統計：

```txt

contenteditable
input[type="text"]
input[type="search"]
```

### 8.2 禁止統計的欄位

不得統計以下欄位：

```txt
input[type="password"]
input[type="email"]
input[type="tel"]
input[type="number"]
input[type="file"]
input[type="hidden"]
input[type="checkbox"]

input[type="button"]
input[readonly]
input[disabled]

textarea[disabled]
```

### 8.3 不得保存使用者輸入內容

Typetchi 第一版只記錄數量，不記錄內容。

允許保存：

```txt

等級
角色階段

```

禁止保存：

```txt

剪貼簿內容

密碼
Email
電話
信用卡資訊
網址完整 query string
```

## 9. 中文輸入法 Composition 處理

為避免中文輸入法組字期間重複計算，請加入 composition 狀態處理。

建議：

```ts
let isComposing = false;

document.addEventListener('compositionstart', () => {
  isComposing = true;
}, true);

document.addEventListener('compositionend', () => {
  isComposing = false;
}, true);

document.addEventListener('input', (event) => {
  if (isComposing) return;
  handleInput(event);
}, true);
```

v0.1.1 只需避免明顯重複計算，不要求完全精準處理所有輸入法情境。

## 10. 儲存機制規範

### 10.1 不使用宿主頁面的 localStorage

Typetchi 不得使用：

```ts
window.localStorage
window.sessionStorage
document.cookie
```

原因：

```txt
localStorage 屬於目前網站 origin，可能污染宿主網站資料。
Typetchi 的資料應屬於 Chrome Extension 本身。
```

### 10.2 使用 chrome.storage.local

所有 Typetchi 狀態請統一使用：

```ts
chrome.storage.local
```

需要保存：

```txt
totalExp
level
currentStage
todayTypedCount
lastActiveDate
widget x
widget y
widget width
widget height
pinned
collapsed
settings
```

## 11. Storage 封裝

請建立 Storage Wrapper，不要讓 React components 直接呼叫 `chrome.storage.local`。

建議檔案：

```txt
src/storage/extensionStorage.ts
src/storage/petStorage.ts
src/storage/widgetStorage.ts
```

### 11.1 extensionStorage.ts

```ts
export async function getStorageValue<T>(
  key: string,
  defaultValue: T
): Promise<T> {
  const result = await chrome.storage.local.get(key);
  return result[key] ?? defaultValue;
}

export async function setStorageValue<T>(
  key: string,
  value: T
): Promise<void> {
  await chrome.storage.local.set({
    [key]: value,
  });
}
```

### 11.2 建議 Storage Schema

```ts
export interface TypetchiStorageSchema {
  petState: {
    totalExp: number;
    level: number;
    currentStage: 'stage_1' | 'stage_2' | 'stage_3';
    todayTypedCount: number;
    lastActiveDate: string;
  };

  widgetState: {
    x: number;
    y: number;
    width: number;
    height: number;
    pinned: boolean;
    collapsed: boolean;
  };

  settings: {
    enableTypingTracking: boolean;
    enablePasteExp: boolean;
  };
}
```

## 12. Storage 寫入節流

打字時不得每輸入一個字就立即寫入 `chrome.storage.local`。

請使用 debounce 或 batch update。

建議策略：

```txt
React state 即時更新
Storage 每 1000ms 或 2000ms 寫入一次
頁面 visibilitychange 時 flush
頁面 beforeunload 時嘗試 flush
```

範例概念：

```ts
let flushTimer: number | undefined;

function scheduleStorageFlush() {
  if (flushTimer) {
    window.clearTimeout(flushTimer);
  }

  flushTimer = window.setTimeout(() => {
    flushPetState();
  }, 1000);
}
```

## 13. Message Listener 修正

請檢查所有：

```ts
chrome.runtime.onMessage.addListener(...)
```

若 listener 內有 `return true`，就必須保證最後一定會呼叫 `sendResponse`。

若該 listener 不需要非同步回應，請不要 `return true`。

建議統一格式：

```ts
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  Promise.resolve()
    .then(() => handleMessage(message, sender))
    .then((data) => {
      sendResponse({ ok: true, data });
    })
    .catch((error) => {
      sendResponse({
        ok: false,
        error: error instanceof Error ? error.message : String(error),
      });
    });

  return true;
});
```

## 18. 驗收條件

v0.1.1 完成後需符合以下條件：

### 18.1 注入穩定性

1. 重新整理普通網頁後，右下角小視窗穩定出現。
2. 小視窗不會閃一下後消失。
3. Console 不再出現 React error #418。
4. Console 不再出現 message channel closed error。
5. 同一頁面只會存在一個 `#typetchi-root`。
6. SPA 頁面切換時不會重複注入多個 root。

### 18.2 不干擾頁面

1. 普通 input 可以正常輸入。
2. textarea 可以正常輸入。
3. contenteditable 可以正常輸入。
4. 網頁按鈕可以正常點擊。
5. 網頁連結可以正常點擊。
6. 頁面可以正常滾動。
7. 頁面快捷鍵不被 Typetchi 攔截。
8. Typetchi 透明區域不會擋住頁面點擊。
9. Typetchi 不修改宿主頁面既有 DOM。
10. Typetchi CSS 不影響宿主頁面。

### 18.3 隱私與安全

1. 密碼欄位不會增加 EXP。
2. Email 欄位不會增加 EXP。
3. 電話欄位不會增加 EXP。
4. hidden / disabled / readonly 欄位不會增加 EXP。
5. 不保存使用者實際輸入文字。
6. 不使用網站 localStorage。
7. 不使用網站 sessionStorage。
8. 不使用 document.cookie。

### 18.4 儲存功能

1. 使用 `chrome.storage.local` 保存資料。
2. 重新整理後 EXP 仍存在。
3. 重新整理後角色階段仍存在。
4. 重新整理後視窗位置仍存在。
5. 重新整理後視窗尺寸仍存在。
6. 儲存寫入有 debounce / batch，避免每打一個字就寫入。

### 18.5 原功能保留

1. 小視窗仍可拖拉。
2. 小視窗仍可縮放。
3. 小視窗仍可固定。
4. 小視窗仍可解除固定。
5. 打字仍可增加 EXP。
6. 達到門檻仍可進化。
7. UI 仍可正常顯示目前 EXP、level、stage。

## 19. 建議 Codex 執行順序

請依照以下順序處理：

```txt
1. 檢查 React 掛載方式，移除 hydrateRoot，改用 createRoot。
2. 建立或修正 injectRoot.ts。
3. 加入防重複注入。
4. 加入 document.body ready 檢查。
5. 導入 Shadow DOM。
6. 修正 root / widget pointer-events。
7. 檢查是否有污染宿主 DOM / CSS 的行為。
8. 統一 storage 使用 chrome.storage.local。
9. 建立 storage wrapper。
10. 加入 storage debounce。
11. 修正 message listener。
12. 加入 debug log。
13. 回歸測試拖拉、縮放、固定、EXP、進化。
14. 測試不同網站下是否穩定。
```

## 20. 版本完成定義

當 v0.1.1 完成時，Typetchi 應該達成：

```txt
可以穩定注入到一般網頁
小視窗不會因網站 CSS 或 React 掛載問題消失
不影響宿主網站原本操作
不保存使用者輸入內容
不使用宿主網站 localStorage
所有 Typetchi 狀態統一存在 chrome.storage.local
```

此版本完成後，才進入 v0.2.0 的角色動畫與互動體驗優化。
