# Typetchi v0.6.2 Bugfix：Close 行為與 Collapsed Handle 拖曳

## 1. 版本目標

修正 Widget 關閉與收合行為。

## 2. 開發方向

- Close 必須讓整個 Widget 視覺消失，不留下 handle。
- Collapse 保留小 icon / handle。
- Collapsed handle 需要可拖曳。
- Popup 需能將 closed 狀態恢復為顯示。

## 3. 需要檢查的範圍

- PetWidget header actions
- WidgetState closed / collapsed
- popup 顯示 Typetchi 流程
- storage.onChanged 跨分頁同步
- collapsed handle pointer events / drag 行為
- scripts/content-dist.js 是否需要同步

## 4. 注意事項

- close 與 collapse 不可混用。
- close 後 runtime / storage listener 不應停掉。
- collapsed handle 拖曳不可誤觸展開。
- 不要改 evolution system。
- 不要改 EXP / CPM / paste 規則。

## 5. 測試方向

- 完整視窗點 close。
- 完整視窗點 collapse。
- collapsed handle 點擊展開。
- collapsed handle 拖曳。
- popup 重新顯示。
- A/B 分頁同步。
- Chrome extension reload 後狀態正常。

## 6. 驗收條件

- 點 close 後畫面完全沒有 Typetchi icon / handle。
- 點 extension popup 可重新顯示 Typetchi。
- 點 collapse 後保留 handle。
- handle 可拖曳並保存位置。
- 拖曳 handle 不會觸發展開。
- 跨分頁同步正常。
- typecheck / build / check:version 通過。

補充 v0.6.2 close 行為：

關閉不是收合。點擊「關閉」後，Typetchi 必須從瀏覽器頁面上完全消失，不留下 handle / icon / bubble，但資料仍保存在 chrome.storage.local。

使用者之後需要能透過點擊 Chrome 擴充功能圖示開啟 popup，並在 popup 中點擊「顯示 Typetchi」重新顯示 Widget。重新顯示時不得清除 EXP、Level、Stage、統計、位置與尺寸。

請確認 manifest action popup、popup build、storage 寫入、storage.onChanged 跨分頁同步都正常。
