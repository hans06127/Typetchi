# Typetchi v0.4.0 規格：設定、重置功能與版面穩定修正

## 1. 版本目標

v0.4.0 是 Typetchi 的設定與版面穩定修正版。

目前已完成：

* Chrome Extension 基礎架構
* Content Script 注入
* Shadow DOM 隔離
* 不干擾宿主頁面
* chrome.storage.local 儲存
* Extension context invalidated 防護
* 大型 SPA 相容性
* Widget 拖拉、縮放、固定、收合
* 角色動畫與互動回饋
* EXP、Level、三階段進化
* 今日輸入統計
* CPM / WPM 統計
* 貼上文字不計入 EXP

本版本主要修正：

1. 重置按鈕點擊後沒有反應。
2. Widget 內容增加後，底部最後一行跑出框。
3. 設定、統計與操作區需要整理，避免後續功能越加越亂。
4. 重置功能需要有明確定義，避免誤刪或刪不乾淨。

---

## 2. 本版本核心任務

v0.4.0 需要完成：

1. 明確定義重置功能。
2. 修正重置按鈕點擊無反應。
3. 新增重置確認流程。
4. 實作完整資料重置。
5. 實作視窗位置重置。
6. 修正 Widget 跑版。
7. Widget 內容區支援滾動或自適應高度。
8. 統一 Widget layout 結構。
9. 整理設定區與危險操作區。
10. 保留所有既有功能，不破壞 v0.3.0 行為。

---

## 3. 本版本不做項目

v0.4.0 不做以下功能：

```txt
不做分支進化
不做多角色選擇
不做任務系統
不做成就系統
不做排行榜
不做雲端同步
不做 AI 對話
不做後端 API
不做完整打字練習模式
```

分支進化可延後到 v0.5.0 或之後。

# 4. 重置功能定義

目前「重置」語意不清楚，因此 v0.4.0 需要拆成兩種重置。

## 4.1 重置視窗位置

用途：

```txt
只把 Widget 位置、大小、固定狀態、收合狀態恢復預設。
不影響角色 EXP、Level、Stage、今日統計、打字速度。
```

重置內容：

```txt
widget x
widget y
widget width
widget height
pinned
collapsed
closed
```

預設值建議：

```ts
export const DEFAULT_WIDGET_STATE = {
  x: null,
  y: null,
  width: 280,
  height: 360,
  pinned: false,
  collapsed: false,
  closed: false,
};
```

若目前使用 `right / bottom` 定位而不是 `x / y`，請依照現有架構恢復右下角預設位置。

---

## 4.2 重置角色進度

用途：

```txt
清空角色成長資料，回到初始狀態。
```

重置內容：

```txt
totalExp = 0
level = 1
currentStage = stage_1
todayTypedCount = 0
todayMaxCpm = 0
todayMaxWpm = 0
lastActiveDate = today
```

不得重置：

```txt
widget position
widget size
pinned state
collapsed state
settings
```

---

## 4.3 全部重置

v0.4.0 可以先不做完整全部重置。

如果要保留按鈕，請明確命名為：

```txt
重置全部資料
```

並需要二次確認。

但 v0.4.0 最低要求是完成：

```txt
重置視窗位置
重置角色進度
```

# 5. 重置按鈕修正

## 5.1 問題

目前點擊重置按鈕沒有反應。

可能原因：

```txt
button 沒有綁定 onClick
onClick 被 overlay 或 pointer-events 擋住
按鈕在 Shadow DOM 內事件沒有正確處理
reset function 沒有實作
reset function 有執行但沒有更新 React state
reset function 有更新 storage 但沒有同步更新畫面 state
reset function 執行失敗但沒有 log
```

---

## 5.2 重置按鈕需求

請確認按鈕具備：

```tsx
<button
  type="button"
  onClick={handleResetWidgetPosition}
>
  重置視窗位置
</button>
```

不要使用沒有行為的 placeholder button。

---

## 5.3 點擊後需要同時更新 React state 與 storage

錯誤做法：

```txt
只更新 storage，但畫面 state 沒更新
只更新 state，但重新整理後失效
```

正確做法：

```txt
1. 產生 default state
2. setReactState(defaultState)
3. saveWidgetState(defaultState)
4. 顯示成功提示
```

範例：

```ts
async function handleResetWidgetPosition() {
  const nextWidgetState = createDefaultWidgetState();

  setWidgetState(nextWidgetState);
  await saveWidgetState(nextWidgetState);

  showMessage('視窗位置已重置');
}
```

---

## 5.4 重置角色進度流程

```ts
async function handleResetPetProgress() {
  const confirmed = window.confirm('確定要重置角色進度嗎？EXP、等級與今日統計會歸零。');

  if (!confirmed) return;

  const nextPetState = createDefaultPetState();

  setPetState(nextPetState);
  await savePetState(nextPetState);

  showMessage('角色進度已重置');
}
```

注意：

如果不想使用 `window.confirm`，可以改用 Widget 內的自製確認 UI。

但 v0.4.0 至少要有確認流程，避免誤觸。

# 7. Storage 與 State 同步

## 7.1 新增 default state factory

請新增或整理：

```txt
src/config/defaultState.ts
```

建議：

```ts
export function createDefaultPetState(): UserPetState {
  return {
    totalExp: 0,
    level: 1,
    currentStage: 'stage_1',
    todayTypedCount: 0,
    lastActiveDate: getTodayKey(),
  };
}

export function createDefaultTypingStatsState(): TypingSpeedState {
  return {
    recentCpm: 0,
    recentWpm: 0,
    todayMaxCpm: 0,
    todayMaxWpm: 0,
    sessionChars: 0,
    sessionStartedAt: null,
    lastTypedAt: null,
  };
}

export function createDefaultWidgetState(): WidgetState {
  return {
    x: null,
    y: null,
    width: 280,
    height: 360,
    pinned: false,
    collapsed: false,
    closed: false,
  };
}
```

若目前型別不完全相同，請依照現有型別調整。

---

## 7.2 新增 reset API

建議新增：

```txt
src/storage/resetStorage.ts
```

內容包含：

```ts
export async function resetWidgetState(): Promise<WidgetState>;
export async function resetPetProgress(): Promise<UserPetState>;
```

行為：

```txt
產生 default state
寫入 chrome.storage.local
回傳 default state 給 React setState 使用
```

# 8. Widget 跑版修正

## 8.1 問題

目前因為 v0.3.0 加入更多統計列，例如：

```txt
今日輸入
目前速度
今日最高
EXP
Level
Stage
訊息泡泡
```

導致底部最後一行跑出 Widget 框外。

---

## 8.2 修正目標

Widget 必須在固定高度內穩定顯示。

需要符合：

```txt
內容不跑出框
底部不被裁切
按鈕不被擠出
收合 handle 不被蓋住
拖拉與縮放控制仍可操作
```

---

## 8.3 Layout 結構建議

請將 Widget 拆成三段：

```txt
.typetchi-widget
  .typetchi-widget__header
  .typetchi-widget__body
  .typetchi-widget__footer
```

### Header

放：

```txt
標題
固定 / 解除固定
收合按鈕
```

Header 高度固定，例如：

```txt
44px
```

### Body

放：

```txt
角色
EXP bar
訊息泡泡
今日統計
速度統計
```

Body 使用：

```css
flex: 1;
min-height: 0;
overflow-y: auto;
```

### Footer

放：

```txt
設定入口
重置視窗位置
重置角色進度
```

Footer 可固定高度或自適應，但不得被擠出。

---

## 8.4 CSS 建議

```scss
.typetchi-widget {
  display: flex;
  flex-direction: column;
  width: 280px;
  height: 360px;
  min-width: 240px;
  min-height: 280px;
  max-width: min(420px, calc(100vw - 32px));
  max-height: min(560px, calc(100vh - 32px));
  overflow: hidden;
}

.typetchi-widget__header {
  flex: 0 0 auto;
}

.typetchi-widget__body {
  flex: 1 1 auto;
  min-height: 0;
  overflow-y: auto;
  overscroll-behavior: contain;
}

.typetchi-widget__footer {
  flex: 0 0 auto;
}
```

---

## 8.5 內容區 scrollbar

若內容超過高度，允許 body 區域出現內部滾動。

不可讓整個 Widget 內容直接超出外框。

```scss
.typetchi-widget__body::-webkit-scrollbar {
  width: 6px;
}
```

Scrollbar 樣式需低調，不要影響療癒風格。

# 11. 收合狀態版面

收合狀態下不應受完整 Widget 內容高度影響。

收合後只顯示：

```txt
handle
小角色頭像 / icon
```

不得顯示完整 stats panel，避免收合後仍然撐高。

---

# 12. Resize 行為修正

如果 Widget 可縮放，需要確認：

```txt
縮小時內容不溢出
最小高度足以顯示 header + handle
body 可滾動
footer 不被擠出
resize 後 width / height 會 normalize
重新整理後不會恢復成超出 viewport 的大小
```

建議限制：

```ts
const MIN_WIDGET_WIDTH = 240;
const MIN_WIDGET_HEIGHT = 280;
const DEFAULT_WIDGET_WIDTH = 280;
const DEFAULT_WIDGET_HEIGHT = 360;
const MAX_WIDGET_WIDTH = Math.min(420, window.innerWidth - 32);
const MAX_WIDGET_HEIGHT = Math.min(560, window.innerHeight - 32);
```

---

# 13. 不干擾頁面規範

v0.4.0 仍需維持：

```txt
不修改宿主頁面既有 DOM
不攔截宿主頁面輸入
不阻止貼上
不阻止快捷鍵
不阻止頁面滾動
不影響按鈕、連結、表單
Typetchi CSS 不影響宿主頁面
Typetchi UI 只存在 Shadow DOM
```

重置按鈕、設定按鈕事件只能影響 Typetchi 自己的 Shadow DOM UI。

# 14. scripts/content-dist.js 注意事項

本專案目前 build 流程會將：

```txt
scripts/content-dist.js
```

複製成：

```txt
dist/content/index.js
```

因此如果 v0.4.0 有修改 content script runtime 行為，例如：

```txt
storage reset
widget state reset
layout injection
root style
shadow host style
```

請務必同步更新：

```txt
src/
scripts/content-dist.js
```

並在執行：

```txt
npm run build
```

後確認：

```txt
dist/content/index.js
```

包含最新修正。

若本次只改 React component 且 build 流程會正常 bundle React component，可不直接手動修改 `scripts/content-dist.js`。

但若目前專案的 content script 實際仍依賴 `scripts/content-dist.js` 手寫輸出，請務必同步。

# 15. 測試案例

## 15.1 重置視窗位置

1. 拖拉 Widget 到左上角。
2. 調整 Widget 大小。
3. 點擊「重置視窗位置」。
4. Widget 回到右下角。
5. Widget 大小回到預設。
6. 重新整理頁面後仍維持預設位置與大小。
7. EXP、Level、Stage 不受影響。

---

## 15.2 重置角色進度

1. 手打文字累積 EXP。
2. 確認 Level / Stage 有變化。
3. 點擊「重置角色進度」。
4. 出現確認提示。
5. 確認後 EXP 變 0。
6. Level 回到 1。
7. Stage 回到 stage_1。
8. todayTypedCount 變 0。
9. todayMaxCpm / todayMaxWpm 變 0。
10. Widget 位置與大小不受影響。

---

## 15.3 重置取消

1. 點擊「重置角色進度」。
2. 在確認提示選擇取消。
3. EXP 不變。
4. Level 不變。
5. Stage 不變。
6. todayTypedCount 不變。

---

## 15.4 跑版測試

1. Widget 使用預設大小 280x360。
2. 顯示角色、EXP、今日輸入、目前速度、今日最高、按鈕。
3. 最底部文字不得跑出框。
4. 若內容過多，body 區塊應可滾動。
5. Header 與 Footer 不應被擠出。
6. 收合 handle 不應被遮住。

---

## 15.5 Resize 測試

1. 將 Widget 縮到最小。
2. 內容不得跑出外框。
3. Body 應可滾動。
4. Footer 按鈕仍可操作。
5. 重新整理後尺寸仍被 normalize 到合法範圍。

# 16. 驗收條件

v0.4.0 完成後需符合：

1. 「重置視窗位置」按鈕點擊後有反應。
2. 「重置視窗位置」會重設 Widget 位置、尺寸、固定、收合狀態。

5. 確認後 EXP 回到 0。
6. 確認後 Level 回到 1。
7. 確認後 Stage 回到 stage_1。
8. 確認後 todayTypedCount 回到 0。
9. 確認後 todayMaxCpm / todayMaxWpm 回到 0。

11. Widget 預設大小下，底部最後一行不得跑出框。
12. 內容過多時，body 區域可滾動。
13. Header 固定在上方，不被內容擠壓。
