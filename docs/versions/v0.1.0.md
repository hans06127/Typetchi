# Chrome Extension Project Plan：療癒打字成長小視窗

## 1. 專案目標

建立一個 Chrome 擴充套件，提供一個可常駐在網頁上的「療癒小視窗」。

使用者在網頁中打字時，擴充套件會監聽有效輸入文字，將輸入量轉換為角色經驗值。角色會依照經驗值逐步成長，初期採三階段進化，未來預留分支進化與特殊條件進化。

第一版目標不是做完整遊戲，而是完成穩定可用的核心系統：

* 可顯示療癒角色小視窗
* 小視窗可拖拉、縮放、固定、解除固定
* 可監聽使用者打字輸入
* 可累積經驗值
* 可依經驗值切換角色階段
* 可保存使用者進度
* 架構預留角色樣式、動畫、進化分支擴充

## 3. 核心使用情境

### 3.1 基本流程

1. 使用者安裝 Chrome 擴充套件
2. 每次開啟網頁時，右下角出現療癒小視窗
3. 使用者可以拖曳小視窗到任意位置
4. 使用者可以縮放小視窗大小
5. 使用者可以固定小視窗位置
6. 使用者在網頁中打字
7. 系統判斷有效輸入文字
8. 有效輸入轉換為經驗值
9. 經驗值累積到門檻後，角色進化
10. 下次開啟網頁時，保留角色狀態與視窗位置

* 建立 Manifest V3
* 建立 Content Script

* 建立 React UI 注入點
* 建立基本資料存取封裝

#### B. 療癒小視窗

小視窗需支援：

* 預設出現在右下角
* 可拖拉
* 可縮放
* 可固定位置
* 可解除固定
* 可關閉 / 收合
* 保存位置與尺寸

#### C. 角色顯示

第一版可以先使用 placeholder 角色，不需要正式美術。

角色狀態：

* Stage 1：幼年期
* Stage 2：成長期
* Stage 3：成熟期

角色顯示內容：

* 角色圖像 / 動畫區
* 等級
* 經驗值
* 下一階段所需經驗
* 今日輸入字數

#### D. 打字監聽

監聽目標：

* input
* textarea
* contenteditable

第一版先排除：

* password input
* hidden input
* disabled input
* readonly input

有效輸入規則：

* 使用 input event 監聽文字變化
* 計算新增字元數
* 只計算實際新增的文字
* 刪除文字不扣經驗
* 貼上文字可以先計算，但未來可做限制
* 中文、英文、數字、符號皆可計算

#### E. 經驗值系統

第一版經驗值規則：

```ts
1 個有效輸入字元 = 1 exp
```

角色階段門檻：

```ts
Stage 1: 0 exp
Stage 2: 500 exp
Stage 3: 2000 exp
```

等級規則第一版可簡化：

```ts
level = Math.floor(totalExp / 100) + 1
```

#### F. 資料保存

需要保存：

* totalExp
* level
* currentStage
* todayTypedCount
* lastActiveDate
* window position
* window size
* pinned state
* collapsed state

使用：

```ts
chrome.storage.local
```

## 8. 角色動畫可行方案

### 8.1 第一版建議

第一版建議使用 CSS animation 或 Sprite Sheet。

原因：

* 實作成本低
* 不需要額外動畫 runtime
* 容易與角色階段切換整合
* 對 Chrome Extension 友善
* 適合 placeholder 階段

### 8.2 可選方案比較

| 方案            | 優點      | 缺點      | 建議       |
| ------------- | ------- | ------- | -------- |
| CSS Animation | 最簡單、效能好 | 動作有限    | v0.1 推薦  |
| GIF / APNG    | 快速可見    | 不易控制狀態  | 可用於測試    |
| Sprite Sheet  | 可控、遊戲感強 | 需要切圖    | v0.2 推薦  |
| Lottie        | 動畫漂亮    | 需注意套件體積 | v0.3 可評估 |
| Canvas        | 彈性最大    | 開發成本高   | 後期再考慮    |

### 8.3 第一版動畫狀態

```ts
export type PetAnimationState = 'idle' | 'happy' | 'level_up' | 'evolve';
```

第一版最少需要：

* idle：待機動畫
* happy：獲得經驗時短暫反應
* evolve：進化時動畫

## 9. 打字監聽設計

### 9.1 監聽策略

Content Script 在頁面中註冊事件：

```ts
document.addEventListener('input', handleInput, true);
```

### 9.2 需要判斷的元素

```ts
function isTrackableInput(target: EventTarget | null): boolean {
  if (!(target instanceof HTMLElement)) return false;

  if (target instanceof HTMLInputElement) {
    const ignoredTypes = ['password', 'hidden', 'checkbox', 'radio', 'file', 'submit', 'button'];
    if (ignoredTypes.includes(target.type)) return false;
    if (target.disabled || target.readOnly) return false;
    return true;
  }

  if (target instanceof HTMLTextAreaElement) {
    if (target.disabled || target.readOnly) return false;
    return true;
  }

  if (target.isContentEditable) {
    return true;
  }

  return false;
}
```

### 9.3 字數計算策略

第一版可以使用 WeakMap 記錄元素上一次文字長度。

```ts
const previousValueMap = new WeakMap<HTMLElement, string>();

function getElementText(element: HTMLElement): string {
  if (element instanceof HTMLInputElement || element instanceof HTMLTextAreaElement) {
    return element.value;
  }

  if (element.isContentEditable) {
    return element.innerText;
  }

  return '';
}

function calculateAddedChars(element: HTMLElement): number {
  const currentValue = getElementText(element);
  const previousValue = previousValueMap.get(element) ?? '';

  previousValueMap.set(element, currentValue);

  const diff = currentValue.length - previousValue.length;

  return Math.max(diff, 0);
}
```

### 9.4 注意事項

第一版先接受下列限制：

* 貼上一大段文字也會計算經驗值
* 部分複雜編輯器可能計算不準
* 中文輸入法組字期間可能需要搭配 composition event 優化
* iframe 內的輸入第一版可先不處理
* Shadow DOM 內的輸入第一版可先不處理

## 10. 經驗值與進化邏輯

### 10.1 經驗值計算

```ts
export function calculateExpFromTyping(addedChars: number): number {
  return addedChars;
}
```

### 10.2 等級計算

```ts
export function calculateLevel(totalExp: number): number {
  return Math.floor(totalExp / 100) + 1;
}
```

### 10.3 階段計算

```ts
export function calculateStage(totalExp: number): PetStageId {
  if (totalExp >= 2000) return 'stage_3';
  if (totalExp >= 500) return 'stage_2';
  return 'stage_1';
}
```

### 10.4 更新流程

```ts
export function applyTypingExp(
  state: UserPetState,
  addedChars: number
): UserPetState {
  const gainedExp = calculateExpFromTyping(addedChars);
  const nextTotalExp = state.totalExp + gainedExp;

  return {
    ...state,
    totalExp: nextTotalExp,
    level: calculateLevel(nextTotalExp),
    currentStage: calculateStage(nextTotalExp),
    todayTypedCount: state.todayTypedCount + addedChars,
  };
}
```

## 12. 權限規劃

第一版 manifest 權限建議：

```json
{
  "permissions": ["storage"],
  "host_permissions": ["<all_urls>"]
}
```

說明：

* storage：保存角色進度與視窗狀態
* host_permissions：讓 content script 可在網頁中注入 UI

若不想一開始要求過大權限，可先限制在特定網站測試。
