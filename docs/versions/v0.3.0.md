# Typetchi v0.3.0 規格：打字能力統計與貼上排除 EXP

## 1. 版本目標

v0.3.0 是 Typetchi 的「打字能力統計」版本。

前面版本已完成：

* Chrome Extension 基礎架構
* Content Script 注入
* Shadow DOM 隔離
* 不干擾宿主頁面
* chrome.storage.local 儲存
* Extension context invalidated 防護
* 大型 SPA 相容性
* Widget 拖拉、縮放、固定、收合
* 角色動畫與互動回饋
* EXP、Level、三階段進化
* 今日輸入統計

本版本要開始區分：

```txt
真正手打輸入
貼上輸入
短時間大量輸入
打字速度統計
```

v0.3.0 的核心重點是：

```txt
貼上的字不能納入經驗值。
```

## 4. 重要規則：貼上不給 EXP

### 4.1 規則

使用者透過貼上產生的文字，不得計入：

```txt
totalExp
todayTypedCount
level
stage progress
typing streak
typing speed
```

也就是：

```txt
貼上文字 = 不給經驗值、不算今日輸入、不算打字速度
```

### 4.2 允許行為

貼上行為本身不得被阻止。

Typetchi 只能旁路偵測，不可干擾使用者貼上。

禁止：

```ts
event.preventDefault();
event.stopPropagation();
```

除非事件發生在 Typetchi 自己的 Shadow DOM UI 內。

## 5. Paste Detection 設計

### 5.1 監聽 paste event

請在 content script 或 typing tracker 中加入：

```ts
document.addEventListener('paste', handlePaste, true);
```

### 5.2 Paste 狀態記錄

當偵測到 paste event 時，記錄該元素短時間內的 paste 狀態。

建議：

```ts
const pasteElementMap = new WeakMap<HTMLElement, number>();

function markElementAsPasted(element: HTMLElement): void {
  pasteElementMap.set(element, Date.now());
}

function wasRecentlyPasted(element: HTMLElement): boolean {
  const pastedAt = pasteElementMap.get(element);
  if (!pastedAt) return false;

  return Date.now() - pastedAt <= 1000;
}
```

### 5.3 Paste 後的 input event 不給 EXP

因為實際文字變化通常會在 `paste` 後觸發 `input`，所以 `input` handler 需要判斷：

```ts
if (wasRecentlyPasted(element)) {
  return;
}
```

## 6. 疑似大量瞬間輸入排除

### 6.1 目的

有些網站或瀏覽器情境可能不會明確觸發 paste event，或 input event 內一次新增大量文字。

因此 v0.3.0 需要加入基本防護：

```txt
單次 input 新增字數過大時，不計入 EXP。
```

### 6.2 建議門檻

```ts
const MAX_CHARS_PER_INPUT_EVENT = 20;
```

如果單次 input event 新增字數超過 20：

```txt
不計入 EXP
不計入 todayTypedCount
不計入 typing speed
```

### 6.3 注意

這是保守規則，不追求完美反作弊。

目的只是避免使用者貼上一大段文字直接升級。

### 7.1 狀態

```ts

```

### 7.2 事件

```ts
document.addEventListener('compositionstart', () => {
  isComposing = true;
}, true);

document.addEventListener('compositionend', () => {
  isComposing = false;
}, true);
```

### 7.3 input handler

```ts

```

v0.3.0 目標：


## 8. 有效輸入定義

### 8.1 可計入 EXP 的輸入

只有符合以下條件的 input 才能計入 EXP：

```txt

```


## 11. Typing Session 統計

### 11.1 目標

建立未來打字練習、任務、成就系統會使用的基礎統計。

### 11.2 Session 定義

如果使用者連續打字，且兩次有效輸入間隔小於 10 秒，視為同一 session。

如果超過 10 秒沒有有效輸入，下一次有效輸入開啟新 session。

```ts
const TYPING_SESSION_IDLE_TIMEOUT = 10_000;
```

### 11.3 Session State

```ts
export interface TypingSessionState {
  sessionStartedAt: number | null;
  lastTypedAt: number | null;
  sessionChars: number;
}
```

## 12. 最近 60 秒打字速度

### 12.1 CPM

CPM = Characters Per Minute。

統計最近 60 秒內的有效手打字元數。

```ts
export function calculateCpm(events: TypingInputEvent[], now: number): number {
  const windowStartedAt = now - 60_000;

  return events
    .filter((event) => event.timestamp >= windowStartedAt)
    .reduce((sum, event) => sum + event.addedChars, 0);
}
```

### 12.2 WPM

WPM = Words Per Minute。

中文環境下很難用英文 word 精準定義，因此 v0.3.0 採通用估算：

```txt
WPM = CPM / 5
```

```ts
export function calculateWpm(cpm: number): number {
  return Math.round(cpm / 5);
}
```

## 14. 最近事件保存策略

為了計算最近 60 秒 CPM，不需要永久保存全部事件。

請只在 runtime memory 中保留最近 60 秒到 120 秒的有效 typing events。

建議：

```ts
const TYPING_EVENT_RETENTION_MS = 120_000;
```

每次新增事件後清理舊事件：

```ts
function pruneTypingEvents(events: TypingInputEvent[], now: number): TypingInputEvent[] {
  const threshold = now - TYPING_EVENT_RETENTION_MS;
  return events.filter((event) => event.timestamp >= threshold);
}
```

注意：

```txt
typing events 不需要持久化到 chrome.storage.local。
```

## 23. 隱私規範

v0.3.0 必須維持以下原則：

```txt
只統計字數，不保存文字內容。
只統計時間戳，不保存欄位內容。
只記錄速度，不保存貼上的內容。
不讀取 clipboardData 的文字內容。
不保存完整 URL query string。
不傳送任何資料到外部伺服器。
```

### 23.1 paste event 注意

偵測 paste 時，不需要讀取：

```ts
event.clipboardData?.getData('text')
```

只需要知道發生 paste 行為即可。

### 26.1 手打測試

### 26.2 貼上測試

2. 貼到 textarea。
3. totalExp 不增加。
4. todayTypedCount 不增加。
5. recentCpm 不增加。

### 26.3 大量瞬間輸入測試

1. 讓單次 input event 新增超過 20 字。
2. 不增加 EXP。
3. 不增加 todayTypedCount。
4. 不增加 recentCpm。

### 26.4 中文輸入法測試

1. 使用中文輸入法輸入文字。
2. composition 中間態不重複計算。
3. 最終確認輸入後才計算有效字數。

### 26.5 欄位安全測試

以下欄位輸入不得增加 EXP：

```txt

email
tel
number
hidden
readonly
disabled
```

### 26.6 Storage 測試

1. 重新整理頁面後 todayMaxCpm / todayMaxWpm 保留。
2. 日期變更後 todayTypedCount / todayMaxCpm / todayMaxWpm 重置。
3. totalExp / level / currentStage 不重置。
4. extension reload 後舊頁面不得持續噴 unhandled error。
