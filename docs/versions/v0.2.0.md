# Typetchi v0.2.0 規格：角色動畫與互動體驗強化

## 1. 版本目標

v0.2.0 是 Typetchi 的互動體驗強化版。

前面版本已完成：

* Chrome Extension 基礎架構
* Content Script 注入
* Shadow DOM 隔離
* 不干擾宿主頁面
* chrome.storage.local 儲存
* Extension context invalidated 防護
* Widget 顯示穩定性
* 大型 SPA 相容性
* 小視窗拖拉、縮放、固定、收合
* 基礎打字 EXP 與三階段進化

本版本開始強化 Typetchi 的「療癒感」與「角色生命感」。

---

## 2. 本版本核心任務

v0.2.0 主要完成以下功能：

1. 角色待機動畫
2. 打字時角色反應動畫
3. EXP 增加視覺回饋
4. Level up 動畫
5. 進化動畫
6. 今日輸入統計顯示
7. 基礎訊息泡泡
8. 收合 / 展開體驗細節優化
9. 動畫狀態機整理
10. 保持不干擾宿主網站

## 4. 角色動畫系統

### 4.1 目標

讓角色不只是靜態圖片，而是具有基本生命感。

第一版動畫可以先使用 CSS animation，不需要導入 Lottie、Canvas 或複雜動畫引擎。

---

## 5. 動畫狀態

請建立角色動畫狀態：

```ts
export type PetAnimationState =
  | 'idle'
  | 'typing'
  | 'happy'
  | 'level_up'
  | 'evolve';
```

### 狀態說明

| 狀態       | 觸發條件        | 說明            |
| -------- | ----------- | ------------- |
| idle     | 預設狀態        | 角色輕微浮動、呼吸感    |
| typing   | 使用者正在打字     | 角色有反應，例如跳動、點頭 |
| happy    | EXP 增加後短暫觸發 | 表示收到成長能量      |
| level_up | 等級提升        | 顯示升級動畫        |
| evolve   | 階段進化        | 顯示進化動畫        |

---

## 6. 動畫優先權

若多個動畫同時觸發，請依照以下優先順序：

```txt
evolve > level_up > happy > typing > idle
```

例如：

* 如果打字同時造成 level up，應播放 `level_up`
* 如果打字同時造成進化，應播放 `evolve`
* evolve 播放時，不應被 typing 動畫打斷

---

## 7. 動畫時間建議

```ts
export const PET_ANIMATION_DURATION = {
  typing: 400,
  happy: 800,
  level_up: 1200,
  evolve: 1800,
} as const;
```

## 12. 打字反應動畫

### 12.1 目標

使用者打字時，Typetchi 角色要有短暫反應。

### 12.2 行為

當 `typing tracked` 發生且 `addedChars > 0`：

```txt
角色進入 typing 狀態
400ms 後回到 idle
```

但如果期間觸發 level up 或 evolve，應改播高優先權動畫。

## 14. Level Up 動畫

### 14.1 觸發條件

當打字前後 level 發生變化：

```ts
previousLevel < nextLevel
```

觸發 `level_up` 動畫。

### 14.2 UI 顯示

顯示訊息泡泡：

```txt
Level up!
```

或中文：

```txt
升級了！
```

### 14.3 注意事項

若同時發生 stage 進化，優先播放 evolve，不重複播放 level_up。

---

## 15. 進化動畫

### 15.1 觸發條件

當打字前後 currentStage 發生變化：

```ts
previousStage !== nextStage
```

觸發 `evolve` 動畫。

### 15.2 UI 顯示

顯示訊息泡泡：

```txt
進化了！
```

### 15.3 動畫流程

建議：

```txt
1. 先播放 evolve 動畫
2. 動畫中途或結束後切換 stage 視覺
3. 顯示「進化了！」訊息
```

v0.2.0 可簡化為：

```txt
更新 stage 後直接播放 evolve 動畫
```

## 20. EXP Bar 優化

### 20.1 顯示內容

EXP Bar 顯示目前階段進度，而不是總 EXP 直接除固定值。

例如：

```txt
Stage 1 → Stage 2: 420 / 500
Stage 2 → Stage 3: 800 / 1500
Stage 3: Max Stage
```

### 20.2 建議函式

```ts
export interface StageProgress {
  current: number;
  required: number;
  percentage: number;
  isMaxStage: boolean;
}

export function calculateStageProgress(totalExp: number): StageProgress {
  if (totalExp < 500) {
    return {
      current: totalExp,
      required: 500,
      percentage: Math.min((totalExp / 500) * 100, 100),
      isMaxStage: false,
    };
  }

  if (totalExp < 2000) {
    return {
      current: totalExp - 500,
      required: 1500,
      percentage: Math.min(((totalExp - 500) / 1500) * 100, 100),
      isMaxStage: false,
    };
  }

  return {
    current: 2000,
    required: 2000,
    percentage: 100,
    isMaxStage: true,
  };
}
```

## 23. 狀態更新流程

當打字事件發生：

```txt
1. typing tracker 偵測 addedChars
2. expSystem 計算 gainedExp
3. petState 更新 totalExp / level / stage / todayTypedCount
4. 比較 previousState 與 nextState
5. 若 stage 改變 → animationState = evolve
6. 否則若 level 改變 → animationState = level_up
7. 否則 → animationState = typing 或 happy
8. 顯示 +EXP toast
9. 視需要顯示 speech bubble
10. debounce 保存 storage
```

## 25. 注意 scripts/content-dist.js

本專案目前 build 流程會將：

```txt
scripts/content-dist.js
```

複製成：

```txt
dist/content/index.js
```

因此如果 v0.2.0 的功能有任何 content script runtime 行為，請確認：

```txt
src/ 版本有更新
scripts/content-dist.js 有同步更新
npm run build 後 dist/content/index.js 有實際更新
```

若本次只改 React component 且 build 流程會正常 bundle React component，可不直接手動修改 `scripts/content-dist.js`。

但若目前專案的 content script 實際仍依賴 `scripts/content-dist.js` 手寫輸出，請務必同步。

## 26. 驗收條件

### 26.1 角色動畫

1. Widget 顯示角色。
2. 角色在 idle 狀態有輕微動畫。
3. 打字時角色會短暫反應。
4. level up 時播放升級動畫。
5. stage 改變時播放進化動畫。
6. evolve 優先權高於 level_up。
7. 動畫結束後回到 idle。

### 26.2 EXP 回饋

1. 打字增加 EXP 時顯示 `+N EXP`。
2. EXP toast 不會永久停留。
3. 快速打字時不會產生大量重疊 toast。
4. EXP bar 會隨 EXP 增加更新。
5. EXP bar 顯示目前階段進度。
6. Max stage 時顯示已達最高階段或類似文案。

### 26.3 訊息泡泡

1. 打字時偶爾顯示療癒訊息。
2. level up 時顯示升級訊息。
3. evolve 時顯示進化訊息。
4. typing message 不應過度頻繁。
5. 訊息泡泡不影響拖拉、縮放、輸入。

### 26.4 今日統計

1. Widget 顯示今日輸入字數。
2. 日期變更時 todayTypedCount 歸零。
3. totalExp 不會因日期變更歸零。
4. level 不會因日期變更歸零。
5. currentStage 不會因日期變更歸零。

### 26.5 收合 / 展開

1. 收合時顯示 handle 或小頭像。
2. 收合方向維持向下收合。
3. 點擊 handle 可展開。
4. 收合狀態重新整理後仍保留。
5. 收合狀態不擋住宿主頁面主要操作。

### 26.6 不干擾頁面

1. 不修改宿主頁面既有 DOM。
2. 不攔截宿主頁面 input。
3. 不阻止頁面快捷鍵。
4. 不保存使用者輸入內容。
5. 不讀取 password / email / tel 等欄位。
6. ChatGPT / GitHub / YouTube / 一般表單頁仍可正常操作。

### 26.7 開發檢查

1. `npm run typecheck` 通過。
2. `npm run build` 通過。
3. build 後 extension 可正常載入。
4. reload extension 後，舊頁面不應持續噴 unhandled error。
5. 重新整理頁面後 storage 狀態正常恢復。
