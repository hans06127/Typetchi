# Typetchi v0.7.0：任務系統完整版

## 1. 版本目標

v0.7.0 的目標是加入 Typetchi 的任務系統，讓使用者不只是被動累積 EXP，而是每天有明確的小目標可以完成。

本版要讓 Typetchi 具備：

* 每日任務
* 任務進度
* 任務完成狀態
* 任務獎勵 EXP
* 任務完成提示
* 任務每日重置
* 任務跨分頁同步
* 任務與既有 EXP / Level / Evolution System 整合

v0.7.0 完成後，Typetchi 應從「打字統計 + 寵物成長」進一步變成「有每日目標的療癒打字夥伴」。

## 3. 本版本核心需求

v0.7.0 需要完成以下項目：

1. 新增每日任務資料結構。
2. 新增每日任務產生或初始化流程。
3. 支援多種任務類型。
4. 任務進度會根據有效手打輸入更新。
5. 貼上文字不得推進任務。
6. 任務完成後給予 bonus EXP。
7. 任務獎勵不得重複領取。
8. 任務完成後顯示提示。
9. 任務狀態需寫入 `chrome.storage.local`。
10. 任務狀態需跨分頁同步。
11. 日期變更後每日任務需重置。
12. 任務獎勵 EXP 需走既有 EXP / Level / Evolution 流程。
13. Widget 需顯示今日任務區塊。
14. 不破壞既有 Widget close / collapse / popup reopen 行為。
15. 不破壞 v0.6.x evolution system。

## 5. 任務類型

v0.7.0 至少支援以下任務類型。

### 5.1 手打字數任務

範例：

* 今日手打 300 字
* 今日手打 1000 字

進度來源：

* 有效手打字元數
* 不包含貼上文字
* 不包含疑似大量瞬間輸入
* 不包含敏感欄位輸入

---

### 5.2 速度任務

範例：

* 今日最高速度達到 60 CPM
* 今日最高速度達到 100 CPM

進度來源：

* 既有 `todayMaxCpm`
* 或專案目前使用的今日最高速度 state

注意：

* 貼上文字不得影響速度任務
* 大量瞬間輸入不得影響速度任務
* 若速度統計目前只存在 runtime，請確認今日最高速度有持久化並可跨分頁同步

---

### 5.3 打字 Session 任務

範例：

* 今日完成 3 次有效打字 session
* 今日完成 1 次連續輸入 session

基本定義：

* 一段連續有效手打輸入可視為一個 session
* 兩次有效輸入間隔超過既有 idle timeout，可視為新的 session
* 貼上文字不得建立 session
* 無效輸入不得建立 session

如果目前已有 session state，請沿用既有邏輯。
如果尚未完整實作 session 統計，v0.7.0 可先採簡化版：

* 有效手打開始時建立 session
* 超過 idle timeout 後，下一次有效輸入建立新 session
* 每日 session count 可推進任務

## 6. 任務資料設計方向

任務資料至少需要支援：

* 任務 ID
* 任務類型
* 任務標題
* 任務描述
* 目標值
* 目前進度
* 是否完成
* 是否已領取獎勵
* 獎勵 EXP
* 完成時間
* 日期 key

建議任務狀態分成：

* 任務定義：任務是什麼
* 今日任務狀態：今天進度與完成狀態

避免未來任務越多時，所有資料混在一起難以維護。

---

## 7. 每日任務初始化

每日第一次載入 Typetchi 時，需要確認今日任務是否存在。

流程方向：

1. 讀取目前日期 key。
2. 從 storage 讀取 daily missions。
3. 如果沒有今日任務，建立今日任務。
4. 如果 storage 裡是舊日期任務，重置為今日任務。
5. 如果是今日任務，沿用目前進度。
6. 初始化後同步到 React state。
7. 若有 migration 或重置，需保存回 storage。

v0.7.0 可以先使用固定任務組，不需要隨機任務。

建議固定三個：

* 今日手打 300 字
* 今日最高速度達到 60 CPM
* 今日完成 3 次打字 session

---

## 8. 任務進度更新規則

### 8.1 有效手打輸入

當有效手打輸入發生時，應更新：

* 今日輸入字數
* EXP
* CPM / WPM
* 今日最高速度
* 相關每日任務進度

任務進度更新後，如果達標：

* 標記任務完成
* 發放 bonus EXP
* 顯示任務完成提示
* 保存 storage
* 跨分頁同步

---

### 8.2 貼上文字

貼上文字不得：

* 增加 EXP
* 增加 todayTypedCount
* 增加 CPM / WPM
* 推進任務
* 完成任務
* 建立打字 session
* 觸發任務獎勵

貼上後可以保留既有提示，例如：

* 貼上的文字不會增加經驗值
* 只計算手打的文字喔

但不需要新增更強提示。

---

### 8.3 任務獎勵

任務完成後發放 bonus EXP。

要求：

* 每個任務每日只能發放一次獎勵
* 重新整理頁面不得重複發放
* 跨分頁同步不得重複發放
* 多個分頁同時完成任務時，不得造成明顯重複獎勵

獎勵 EXP 必須走既有成長流程：

任務完成
→ bonus EXP
→ totalExp 更新
→ level 更新
→ evolution system 判斷
→ 可能觸發 level up / evolve
→ storage 保存
→ 跨分頁同步

不得另外寫一套獨立 EXP 邏輯。

---

## 9. 建議任務獎勵

v0.7.0 可先使用固定值：

* 今日手打 300 字：+30 EXP
* 今日最高速度達到 60 CPM：+50 EXP
* 今日完成 3 次打字 session：+40 EXP

後續可再依平衡性調整。

## 10. Widget UI 需求

Widget 內需要新增今日任務區塊。

### 10.1 顯示內容

至少顯示：

* 任務標題
* 任務進度
* 是否完成
* 獎勵 EXP

範例：

* 今日手打 300 字：128 / 300
* 最高速度 60 CPM：42 / 60
* 完成 3 次打字：1 / 3

完成後可顯示：

* 已完成
* +30 EXP
* 勾選狀態

### 10.2 UI 範圍

* 不跑版
* 不擠出 Widget 底部
* 可在 body 區域滾動
* 不影響 header / footer 操作
* 不影響 collapse / close
* 不讓 Widget 遮擋宿主頁面大範圍操作

完整 UI polish 留到 v0.8.0。

## 11. 任務完成提示

任務完成時可以顯示：

* 任務完成！+30 EXP
* 今天也很努力呢！
* Typetchi 收到新的文字能量！

提示要求：

* 不要過度頻繁
* 不要長時間遮住主要操作
* 跨分頁同步時，避免每個分頁都重複跳大量提示
* 同一任務不要重複提示多次

如果實作複雜，v0.7.0 可先只在觸發任務完成的分頁顯示提示。

---

## 12. Storage 與跨分頁同步

### 12.1 Storage

新增或使用既有 storage key 保存每日任務狀態。

資料需包含：

* dateKey
* missions
* progress
* completed
* rewardClaimed
* completedAt
* updatedAt

### 12.2 跨分頁同步

任務狀態需納入 v0.5.0 的跨分頁同步機制。

情境：

* A 分頁完成任務
* B 分頁不重新整理，也看到任務完成
* B 分頁不得再次發放同一任務獎勵

---

## 13. 日期重置

日期變更時需要重置每日任務。

重置內容：

* 任務進度
* 任務完成狀態
* 任務獎勵領取狀態
* dateKey

不得重置：

* totalExp
* level
* currentStage
* currentEvolutionNodeId
* widget position
* widget size
* settings
* 累積成長資料

請沿用目前專案已有的 dateKey / lastActiveDate 處理方式，避免新增不一致的日期判斷。

---

## 14. 與 Evolution System 的關係

v0.7.0 不需要新增任務型進化分支。

但任務獎勵 EXP 可能導致：

* Level up
* Stage evolution
* Evolution node change

因此任務獎勵必須使用既有 evolution system。

未來版本可再將任務完成數作為 evolution condition，但 v0.7.0 只需預留，不必真的啟用任務型進化。

## 16. 測試方向

### 16.1 手打字數任務

測試流程：

1. 重置或建立今日任務。
2. 手打 10 字。
3. 檢查手打字數任務進度增加 10。
4. 繼續手打直到達標。
5. 任務標記完成。
6. bonus EXP 發放一次。

---

### 16.2 貼上不推進任務

測試流程：

1. 複製一段文字。
2. 貼到 textarea 或 contenteditable。
3. 檢查 EXP 不增加。
4. 檢查今日輸入不增加。
5. 檢查任務進度不增加。
6. 檢查任務不會完成。

---

### 16.3 速度任務

測試流程：

1. 手打讓 todayMaxCpm 達到任務目標。
2. 速度任務標記完成。
3. bonus EXP 發放一次。
4. 重新整理後仍是完成狀態。
5. 不重複發放 reward。

---

### 16.4 Session 任務

測試流程：

1. 開始有效手打。
2. 建立或推進 session。
3. 超過 idle timeout 後再次手打，建立新 session。
4. session 任務進度增加。
5. 達標後完成任務並發放 reward。

---

### 16.5 任務獎勵整合

測試流程：

1. 讓任務完成。
2. 檢查 totalExp 增加 rewardExp。
3. 檢查 level 是否依既有規則更新。
4. 若達進化條件，evolution system 正常觸發。
5. 不應出現 reward 重複發放。

---

### 16.6 跨分頁同步

測試流程：

1. 開啟 A 分頁與 B 分頁。
2. A 分頁推進任務。
3. B 分頁不重新整理，也同步任務進度。
4. A 分頁完成任務。
5. B 分頁同步完成狀態。
6. B 分頁不重複領取 reward。

---

### 16.7 日期重置

測試流程：

1. 建立今日任務並推進進度。
2. 模擬日期變更或修改 dateKey。
3. 重新載入 Typetchi。
4. 今日任務進度重置。
5. 任務完成狀態重置。
6. totalExp / level / stage 不重置。

---

### 16.8 UI 回歸

測試流程：

1. Widget 預設大小下顯示任務區。
2. 任務區不得跑出版面。
3. body 可滾動。
4. header / footer 不被擠出。
5. collapse 正常。
6. close 正常。
7. popup reopen 正常。
8. collapsed handle drag 正常。

---

## 17. 驗收條件

v0.7.0 完成後需符合：

1. 每日任務會初始化。
2. 每日任務有 dateKey。
3. 至少支援手打字數任務。
4. 至少支援速度任務。
5. 至少支援 session 任務。
6. 手打文字會推進相關任務。
7. 貼上文字不推進任何任務。
8. 任務達標後會標記完成。
9. 任務完成後會發放 bonus EXP。
10. 每個任務每日 reward 只會發放一次。
11. 重新整理頁面不會重複發放 reward。
12. 跨分頁同步不會重複發放 reward。
13. 任務獎勵 EXP 會走既有 EXP / Level / Evolution System。
14. 任務完成提示正常。
15. Widget 顯示今日任務區塊。
16. Widget 不跑版。
17. 日期變更後每日任務重置。
18. 日期變更不重置角色成長資料。
19. A 分頁任務進度變更後，B 分頁同步。
20. A 分頁任務完成後，B 分頁同步。
21. close 行為仍是完全消失。
22. popup 重新顯示仍正常。
23. collapsed handle drag 仍正常。
24. 不保存使用者實際輸入內容。
25. 不讀取 clipboard 文字內容。
26. 不影響宿主網站操作。
27. `npm run typecheck` 通過。
28. `npm run build` 通過。
29. `npm run check:version` 通過。
30. 若有 content runtime 修正，`scripts/content-dist.js` 與 `dist/content/index.js` 已同步更新。
