# Typetchi v0.8.0：UI / UX 優化與資訊層級整理

## 1. 版本目標

v0.8.0 的目標是整理 Typetchi 的整體 UI / UX，讓目前已完成的功能看起來更像一個可長期使用的產品，而不是功能持續堆疊後的測試面板。

目前 Typetchi 已具備：

- EXP / Level / Stage
- Evolution System
- Daily Missions
- Typing Stats
- Paste Exclusion
- Cross-tab Sync
- Close / Collapse / Popup Reopen
- Collapsed Handle Drag
- Dev-only 測試工具

v0.8.0 不新增大型功能，重點是：

- 重新整理 Widget 資訊層級
- 優化 Daily Mission 顯示
- 優化 EXP / Level / Stage 顯示
- 收斂 Advanced / Dev Only 區塊
- 優化 Popup UI
- 改善小尺寸、resize、scroll 狀態下的使用體驗
- 保持既有功能穩定

---

## 2. 本版本定位

v0.8.0 是 UI / UX 優化版。

這版不是角色美術版，也不是系統重構版。

核心方向：

- 資訊更清楚
- 操作更直覺
- 版面更穩定
- Debug / Advanced 不干擾一般使用
- 任務與成長資訊更像產品功能
- Widget 在小尺寸下仍可讀、可操作

---

## 3. 本版本不做項目

v0.8.0 不做以下項目：

- 不做狐狸角色美術
- 不做 SVG 角色重製
- 不做 PNG / WebP / Asset Renderer
- 不做角色圖鑑
- 不做多角色切換
- 不做成就系統
- 不做排行榜
- 不做雲端同步
- 不做後端 API
- 不重構 Daily Mission System
- 不重構 Evolution System
- 不重構 Storage Schema
- 不改 EXP 計算規則
- 不改貼上排除規則
- 不改 close / collapse / popup reopen 的語意
- 不改 collapsed handle drag 的核心邏輯

如果 UI 調整需要碰到上述功能，請只做必要最小修改，並確保回歸測試通過。

---

## 4. UI 整理原則

v0.8.0 需要讓 Widget 的資訊層級更清楚。

建議資訊優先順序：

1. Header 操作
2. 角色 / 狀態主視覺
3. Level / Stage / EXP
4. 今日任務
5. 今日輸入與打字統計
6. Advanced
7. Dev Only

Dev / Advanced 不應該搶走主要視覺焦點。

---

## 5. Widget Layout 方向

### 5.1 Header

Header 保留：

- Typetchi 標題
- 固定
- 收合
- 關閉

需求：

- Header 不跑版
- 按鈕間距一致
- 小尺寸下仍可點擊
- 固定 / 收合 / 關閉語意不可混淆
- close 仍是完全消失
- collapse 仍保留 collapsed handle

### 5.2 Character Area

角色區是 Widget 的主視覺區。

本版不做角色美術重製，但可以整理角色區版面。

需求：

- 角色顯示區置中
- 角色上下留白合理
- Speech Bubble 不要擋住主要資訊
- Level / Stage 不要重複顯示太多次
- 小尺寸下角色區可以縮小，但不能擠爆下方內容

### 5.3 Progress Area

EXP / Level / Stage 需要整理成更清楚的區塊。

建議顯示：

- Lv. X
- 目前 Stage 名稱
- EXP current / next
- EXP progress bar
- 下一階段提示，如有既有資料可用

需求：

- 不要同時在多個位置重複顯示同一個 Stage 名稱
- EXP bar 不要過厚或過細
- EXP 文字與 bar 對齊清楚
- 滿級 / 最高階段狀態要有合理顯示
- 不硬寫 EXP 門檻，仍沿用既有 Evolution System / Progress logic

### 5.4 Daily Mission Area

v0.7.0 已加入 Daily Missions，v0.8.0 需要優化顯示。

需求：

- 今日任務區塊要比目前更容易閱讀
- 顯示任務名稱
- 顯示進度
- 顯示完成狀態
- 顯示 reward EXP
- 任務完成後有明確但不吵的狀態
- 全部任務完成時可以有簡短提示

建議 UI：

- 任務卡片
- 小型 progress bar
- 完成勾選
- reward badge

注意：

- 只調整 UI，不改任務完成邏輯
- 不改 reward 發放邏輯
- 不讓任務卡片把 Widget 撐爆

### 5.5 Stats Area

今日輸入與打字速度統計需要保持清楚，但不要比任務與 EXP 更搶眼。

可顯示：

- 今日輸入
- Recent CPM / WPM
- Today max CPM / WPM

需求：

- 若空間不足，可簡化顯示
- 不要讓 stats 區塊佔太多高度
- 貼上不計入統計的規則不可改變

### 5.6 Advanced Area

Advanced 區塊目前容易讓 Widget 像 debug panel。

需求：

- Advanced 預設收合，或視覺上降低權重
- 重置角色進度要保留，但不要太顯眼
- 危險操作需維持確認流程
- Advanced 不應影響主要版面閱讀

### 5.7 Dev Only Area

Dev Only 是開發測試工具，不是正式功能。

需求：

- 只在 dev mode 顯示
- 正式模式完全不可見
- 視覺權重降低
- 不要放在主要資訊流中太顯眼的位置
- 保留目前測試用途，例如 Add EXP / Set Stage / 關閉 Dev
- 不得影響一般使用者流程

若目前 dev mode 使用快捷鍵或 localStorage 開啟，請保留並文件化。

## 6. Popup UI 優化

Popup 不應只是一個單純恢復按鈕。

v0.8.0 可以將 popup 整理成輕量狀態面板。

建議顯示：

- Typetchi 狀態
- 顯示 / 隱藏 Widget
- 今日輸入摘要
- 今日任務摘要
- 目前 Level / Stage
- 設定或說明入口，如已有

需求：

- Popup 仍保持輕量
- 不加入大量新功能
- 點擊顯示 Typetchi 後需設定 `closed=false`、`collapsed=false`
- 不破壞 close 後由 popup reopen 的流程

---

## 7. Resize / 小尺寸支援

Widget 可能被使用者縮小，因此 v0.8.0 需要處理小尺寸狀態。

需求：

- 小尺寸下 header 不跑版
- body 可滾動
- footer / advanced 不會擠出邊界
- 任務區可在較小高度下正常閱讀
- 角色區可適度縮小
- 不出現內容溢出到 Widget 外
- resize 後重新整理仍保持合理狀態

如需要，可調整 min-width / min-height，但不要讓 Widget 變得過大。

---

## 8. Scroll 行為

Widget body 需要穩定滾動。

需求：

- Header 固定在上方
- Body 負責滾動
- Footer / resize handle 不被內容擠掉
- Advanced / Dev 區塊在底部仍可操作
- 滾動條不要過度突兀
- 不影響宿主頁面滾動

---

## 9. Toast / Speech Bubble / Feedback

本版可整理提示體驗，但不要改核心事件邏輯。

需求：

- 任務完成提示不過度遮擋
- EXP toast 不要蓋住主要操作太久
- Speech Bubble 不要洗版
- close / collapse / drag 不受提示元素影響
- 跨分頁同步時不要每個分頁都跳大量提示

---

## 10. Reduced Motion / Accessibility

UI 優化時需保留 accessibility。

需求：

- 支援 `prefers-reduced-motion`
- 動畫不要高頻閃爍
- 按鈕要有可理解的 label / title
- 文字對比要足夠
- 小按鈕點擊區域不能太小
- 不要只用顏色表示任務完成狀態

---

## 11. 不可破壞的既有行為

v0.8.0 必須保留以下行為：

### 11.1 Close

- 點擊關閉後，Widget 完全消失
- 不留下 handle / icon / bubble
- 資料仍保留
- 點 Chrome extension popup 可重新顯示

### 11.2 Collapse

- 點擊收合後，保留 collapsed handle
- handle 可點擊展開
- handle 可拖曳
- handle 可拖到 viewport 底部
- 不出現大型模糊 div

### 11.3 Daily Missions

- 手打推進任務
- 貼上不推進任務
- reward 不重複發放
- 日期變更後每日任務重置
- 跨分頁同步正常

### 11.4 EXP / Evolution

- EXP 計算正常
- Level 計算正常
- Evolution System 正常
- 不硬寫新的 stage 門檻
- dev add EXP 仍走既有流程

### 11.5 Privacy

- 不保存使用者實際輸入內容
- 不讀 clipboard 文字內容
- 不統計敏感欄位
- 不新增不必要權限

## 12. 需要檢查的範圍

請優先檢查或修改：

- PetWidget layout
- Header actions
- Character area
- EXP / Level / Stage display
- Daily Mission UI
- Typing Stats UI
- Advanced section
- Dev Only section
- Popup UI
- SpeechBubble / Toast positioning
- Resize handling
- Scroll handling
- CollapsedHandle 回歸測試
- Shadow DOM styles
- scripts/content-dist.js 是否需要同步
- dist/content/index.js build 後是否更新

此專案曾多次出現只改 `src`，但實際 extension runtime 仍跑 `scripts/content-dist.js` / `dist/content/index.js` 的問題。請務必確認 runtime 已同步。

## 13. 測試方向

### 13.1 Widget 預設尺寸測試

流程：

1. 重新 build 並重新載入 extension。
2. 開啟一般網頁。
3. 檢查 Widget 預設狀態。

預期：

- Header 正常
- 角色區正常
- EXP / Level / Stage 清楚
- 今日任務清楚
- Advanced / Dev 不搶主視覺
- 不跑版

### 13.2 小尺寸 / Resize 測試

流程：

1. 將 Widget 縮小到最小允許尺寸。
2. 滾動 body。
3. 展開 / 收合 Advanced。
4. 檢查任務與 EXP 顯示。

預期：

- 內容不溢出
- body 可滾動
- header 不被擠壓
- footer / resize handle 可用
- 任務仍可閱讀

### 13.3 Daily Mission UI 測試

流程：

1. 建立今日任務。
2. 手打推進任務。
3. 完成任務。
4. 完成全部任務。

預期：

- 進度顯示正確
- 完成狀態清楚
- reward 顯示清楚
- 不重複發放 reward
- UI 不跑版

### 13.4 Paste Exclusion 回歸測試

流程：

1. 貼上一段文字。
2. 檢查 EXP。
3. 檢查今日輸入。
4. 檢查任務進度。
5. 檢查 typing animation / feedback。

預期：

- 貼上不增加 EXP
- 貼上不增加今日輸入
- 貼上不推進任務
- 貼上不觸發一般手打回饋

### 13.5 Close / Popup Reopen 測試

流程：

1. 點擊關閉。
2. 確認 Widget 完全消失。
3. 點 Chrome extension icon。
4. 在 popup 中顯示 Typetchi。

預期：

- 關閉後不留下 handle
- popup 可重新顯示
- 重新顯示後資料仍保留

### 13.6 Collapse / Handle Drag 測試

流程：

1. 點擊收合。
2. 拖曳 collapsed handle。
3. 拖到 viewport 底部。
4. 點擊展開。

預期：

- handle 正常顯示
- 可拖曳
- 可拖到底部
- 不彈回中間
- 不出現大型模糊 div
- 點擊可展開

### 13.7 Cross-tab Sync 測試

流程：

1. 開啟 A 分頁與 B 分頁。
2. A 分頁推進任務。
3. A 分頁完成任務。
4. A 分頁 close / collapse。
5. 使用 popup reopen。

預期：

- B 分頁任務狀態同步
- 不重複發放 reward
- close / collapse 狀態同步
- popup reopen 後同步顯示

### 13.8 Popup UI 測試

流程：

1. Widget 顯示時打開 popup。
2. Widget 關閉後打開 popup。
3. 點擊顯示 / 隱藏相關操作。

預期：

- popup 資訊清楚
- 顯示 Typetchi 正常
- 不破壞 storage state
- 不需要重新整理頁面

### 13.9 Large SPA 測試

至少測試：

- ChatGPT
- 一般表單頁
- contenteditable 頁面

預期：

- Widget 不消失
- UI 不受宿主 CSS 影響
- 輸入追蹤正常
- 不干擾宿主網站操作
