# Typetchi v0.6.1

Version: 0.6.1

## v0.6.0 evolution system gap report

The previous v0.6.0 patch did **not** complete the evolution system refactor. It only handled version synchronization and recoverable widget collapse behavior.

Missing items found before this patch:

1. `EvolutionNode` and `EvolutionCondition` existed in types but were not used by the runtime evolution calculation.
2. Evolution data was still a flat stage threshold list only.
3. `calculateStageProgress` still hardcoded `500` and `2000` instead of deriving progress from the configured evolution stages.
4. The fallback `scripts/content-dist.js` path still used hardcoded stage branching and hardcoded progress thresholds.
5. There was no v0.6.1 follow-up document in the repository.

## Completion notes

- Added `evolutionNodes` as the canonical evolution graph/config while preserving `evolutionStages` for display compatibility.
- Added condition evaluation helpers for total EXP, typing speed, daily streak, and special-event conditions.
- Reworked stage calculation to resolve the current stage from unlockable evolution nodes.
- Reworked progress calculation to derive current/required values from the configured stage thresholds.
- Mirrored the same evolution-node calculation path in the dist content script source used by the unpacked extension build.
